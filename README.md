# Cogsworth Depths · 蒸汽深渊

[English README](./README.en.md)

一个**零素材、全程序化生成**的维多利亚瘟疫医生题材 2.5D 轻量肉鸽浏览器游戏。
项目中没有任何一张图片、一个音频文件——模型、纹理、特效、音乐、音效、剧情文本全部由代码实时生成。

## 玩法

清除每个房间的机械守卫(波次遭遇,敌人从预警圈中现身)→ 三选一改装(可多级堆叠、带等级徽章)→ 支线宝箱房 / 齿轮商店 → Boss(50% 血量进入二阶段狂暴)→ 下一层。楼层由种子生成,标题界面可输入种子复现。设置页支持中英双语与音乐/音效独立音量。

## 游戏特色

- **波次遭遇系统**:进房先亮预警圈(≥0.55s),敌人错落现身;2 层起 2 波、4 层起 3 波,Boss 战有专属出场演出
- **10 种蒸汽朋克造物**:发条蜘蛛、哨戒炮、自走锅炉、弹簧跳蚤、分裂球、盾卫、迫击手、钟表狙击手、修补无人机、双 Boss(锅炉魔像 / 人偶剧团长)
- **13 种多级改装**:分裂阀/贯穿/弹射/爆裂弹头/蒸汽护盾/齿轮宠物……等级徽章显示,玩家形态随升级实时变化(加装枪管/烟囱/能量泡)
- **局外成长**:每局结算齿轮残片,在改装间购买永久强化(Hades 夜之镜式),浏览器缓存保存
- **剧情与对话**:每层专属名称与疫医日志横幅,双 Boss 各有出场台词,中英双语全界面
- **场景机关**:周期性蒸汽喷口、宝箱房、齿轮商店,三种房间主题(铁灰/铁锈/铜绿)

## 技术栈与架构

### 双引擎分层渲染

```
┌ PixiJS canvas(透明叠加):HUD / 打击数字 / 粒子 / 升级界面 / 屏幕准星
├ Three.js canvas:3D 场景 / 角色 / 弹幕(主渲染)
└ DOM:标题 / 暂停 / 设置 / 结算
```

游戏逻辑在纯 2D 平面(XZ)运算,3D 只负责表现纵深。两个引擎通过
`Vector3.project(camera)` 世界→屏幕桥接协作:伤害数字、敌人血条、价格标签都"贴"在 3D 世界物体上。

### 程序化建模(零素材)

- **齿轮**:`THREE.Shape` 画齿形轮廓 → `ExtrudeGeometry` 拉伸,互相啮合反向旋转
- **锅炉**:`LatheGeometry` 旋转成型 + 炉膛自发光 + 蒸汽粒子喷口(自定义 GLSL ShaderMaterial,纹理也是 Canvas 代码生成)
- **管线**:`CatmullRomCurve3` + `TubeGeometry` 沿随机折线 + 法兰阀门
- **地面纹理**:Canvas 代码绘制铁板/铆钉/接缝,无图片文件
- **角色**:瘟疫医生人偶(高帽/鸟嘴/护目镜/皮大衣)全由基础几何体拼装,升级后**形态实时变化**(加装枪管/烟囱/能量泡)

### 自研漫画渲染管线(NPR)

渲染链:`RenderPass → 赛璐璐化 → Bloom → 最终调色`

1. **卡通色阶**:`MeshToonMaterial` + 三阶 gradientMap,硬边色块
2. **剪影描边**:背面外扩法(back-face hull),角色轮廓清晰
3. **Fresnel 轮廓光**:`onBeforeCompile` 注入,物体从背景中"跳"出来
4. **Sobel 墨线**:后处理边缘检测
5. **Bayer 有序抖动**:亮度色阶过渡渲染成铜版蚀刻网点
6. **像素化**:块采样对齐物理像素(devicePixelRatio 1.5)
7. **分离色调**:阴影冷青 / 高光暖黄铜的动画式调色

### 程序化音频

- **音效合成器**:振荡器 + 噪声缓冲 + 滤波包络,合成射击/爆炸/金属铿锵/蒸汽嘶鸣;总线挂 WaveShaper 失真 + 压缩器,所有声音统一"脏"质感
- **背景音乐引擎**:tracker 式前瞻调度器(40ms tick,提前 0.15s 排音),112 BPM D 小调 8 小节循环:方波 bass / 琶音 / 合成鼓组(底鼓/军鼓/踩镲)

### 游戏系统

- **波次遭遇**:进房先亮预警圈(≥0.55s),敌人错落现身;2 层起 2 波、4 层起 3 波,波间 0.9s 喘息
- **8 种敌人**:追击/哨戒/自爆/弹簧跳蚤/分裂球/盾卫(正面格挡)/迫击手(抛物线爆破)/Boss(二阶段:螺旋弹幕+召唤)
- **11 种多级升级**:带堆叠上限与等级徽章;齿轮币经济 + 宝箱房 + 商店
- **数值配置表**:`src/game/balance.ts` 集中所有数值,多项式难度曲线(血量 `1+0.22f+0.05f²`,Boss `950+420×层`)

## 性能与手感优化(重点)

### 子步进模拟(Substepping)

低帧率下 dt 变大,子弹会一帧飞跃敌人(穿透),射击手感被帧率绑架。
游戏逻辑把每帧切成 ≤1/90s 的子步执行(最多 6 步):**10fps 下命中判定依然精确**,
手感与帧率完全解耦——这是本项目最重要的一个工程决策。

### 动态分辨率缩放(DRS)

三档画质(1.5x / 1.0x / 0.75x 渲染分辨率),FPS 移动平均 <42 持续 2.5s 自动降档,>56 持续 8s 回升。
**画质优先原则:降档只降分辨率,泛光/描边/网点等风格化效果全档保留。**

### 零热路径分配

每帧执行的瞄准/投影/弹道代码全部复用预分配临时向量,避免 GC 停顿造成的周期性卡顿。

### 正交相机瞄准的坑

`screenToGround` 不能使用 `unproject点 - 相机原点` 作为射线方向——
那只在正中心成立,边缘像素的地面落点会向中心塌缩(曾导致"炮口无视准星")。
正确做法:射线方向恒等于相机朝向,unproject 只用于取射线起点。

### Pointer Lock 虚拟准星

瞄准用 `unadjustedMovement` 原始输入(绕过 OS 鼠标加速),
`mousemove` 事件挂 document 级(部分浏览器锁定后不向元素派发 pointermove),
虚拟光标钳制窗口内,Pixi 准星与光标 1:1 零延迟。

### 其他细节

- 敌人受击 squash & stretch、死亡旋转爆散、击杀冲击波环
- 摆腿动画由**步幅相位**驱动(与位移同步,不吃帧率)
- 场景装饰占用登记系统(`claim()`),程序化摆放不穿模
- 齿轮币磁吸拾取、出生预警 ≥500ms(参考 Hades 预警窗口研究)
