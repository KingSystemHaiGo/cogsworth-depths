# Cogsworth Depths · 后续开发计划 (ROADMAP)

> 目标:从一个可玩 demo 成长为内容完整、可长期维护扩展的蒸汽朋克肉鸽。
> 原则:画质优先、零素材纯程序化、数值与内容集中可配、每步可验证。

## 当前状态 (v0.9)

- 完整肉鸽循环:种子化楼层 → 波次战斗 → 三选一改装 → 宝箱/商店 → 双 Boss → 下一层
- 局外成长:齿轮残片 + 改装间(localStorage)
- 10 种敌人、13 种升级、2 个 Boss、3 种房间主题、蒸汽机关
- 性能:子步进模拟、DRS 动态分辨率、子弹 InstancedMesh、火花 ParticleContainer、shader 预热

---

## Phase 1 · 性能与渲染管线 (地基)

- [x] 子弹 InstancedMesh(384 draw calls → 2)
- [x] 火花 ParticleContainer(每火花 1 draw → 全体 1 draw)
- [x] 房间静态几何合并(墙体+包边 16+ 网格 → 2 次 draw call)
- [x] 敌人网格对象池(避免生成/销毁的 GC 与重新编译)
- [x] Pixi `cacheAsTexture` 与纹理图集:评估后跳过(仅 4 张程序纹理,收益甚微)
- [ ] 楼层生成移入 Web Worker(大地图生成不阻塞主线程)

## Phase 2 · 内容深度 (关卡与敌人)

- [x] 第三 Boss:钟表巨像(时间凝滞领域/钟摆横扫弹幕,按层三位轮换)
- [x] 精英房间:词缀敌人(迅捷/分裂/坚盾),掉落双倍
- [x] 挑战房间:限时清版(成功奖齿轮币+升级,超时无奖)
- [ ] 每层主题深化:下水道(毒水区)/工厂(传送带)/剧场(移动舞台灯)/锅炉之心(熔岩地形)
- [ ] 敌人协同设计:修补无人机+盾卫、迫击手+跳蚤等组合波次

## Phase 3 · 系统与成长 (可玩性)

- [ ] 武器系统:2-3 种基础武器(蒸汽枪/铆钉霰弹/磁轨炮),局内切换
- [ ] 升级协同:弹射+爆裂、宠物+分裂等联动词条
- [ ] 局外成长扩展:更多改装间项目、成就解锁新升级/新武器
- [ ] 每日种子:固定日期种子排行榜(localStorage 纪录)
- [ ] 图鉴:敌人/升级收集册(蒸汽手账风 UI)

## Phase 4 · 表现力 (音画)

- [ ] 过场演出:层间电梯下降过渡、Boss 二阶段场景变化(灯光变红/蒸汽增压)
- [ ] 更多程序化纹理:锈蚀/油污/木纹差异(每房间材质微调)

## Phase 5 · 普适性 (平台)

## 工程规范(长期)

- 数值一律进 `balance.ts`,文案一律进 `i18n.ts`,升级/敌人/Boss 走注册表模式
- 每次改生成逻辑必跑全楼层审计(连通性/预警对应/障碍避让)
- 截图回归:关键界面(playwright)每版本留档
- 构建保持单文件,GitHub Pages 即最新版

## 已知技术债

- 敌人网格每次生成新建,应入池(Phase 1)
- Pixi HUD 每帧重绘 Graphics,量大了要缓存(Phase 1)
- 音乐 pattern 写死在 music.ts,应数据化(Phase 4)
